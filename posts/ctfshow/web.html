  1. ``php <?php include($_GET['url']);?> ``

在网址后面加 ?url=xxx，xxx 会被传递给 include()

php伪协议

PHP 支持一些特殊的“协议”[ http://、file:// 等]

  • php://input

读取 HTTP 请求的 原始 body（请求体），并当作文件内容返回。

如果用在 include 里，body 里有 PHP 代码，就会直接执行。

需要 POST 方式发请求，并且请求体写上代码。

  • php://filter

对文件内容做“过滤”[例如 base64 编码]用来 读取任意文件源码。

不需要 POST，直接 GET 就行。

示例：?url=php://filter/read=convert.base64-encode/resource=flag.php 会把 flag.php 的内容转成 base64 输出，我们解码就看到源码了。

一个完整的 HTTP POST 请求由三部分组成：

  1. 请求行：POST /?url=php://input HTTP/1.1
  2. 请求头部：Host: xxx，Content-Type: ...，Content-Length: ... 等

​ 一个空行（分隔头部和 body）

  1. 请求体：我们写的 PHP 代码
