phps源码泄露有时候能帮上忙

答案（Flag）藏在一个 PHP 文件的“源码备份”里。

在 CTF中，“源码泄露”是一种非常经典的题型。

1. .phps源码泄露

  • .phps文件：

PHP 源代码文件的一种扩展名。

通常，PHP 脚本文件是以 .php结尾的，它们在服务器上运行时，浏览器只能看到执行后的结果（HTML），而看不到背后的代码。

但是，如果服务器配置了特定的规则，当文件以 .phps结尾时，它会将原始的 PHP 源代码以文本形式展示给浏览器。

  • 泄露原因：很多开发者在开发或测试时，为了方便自己查看代码结构，或者为了防止别人直接下载源码，会将一些敏感文件（如配置文件、核心逻辑文件）命名为 .phps。有时服务器配置不当，或者开发者忘记删除这些备份文件，就会导致源码泄露。

2. 结合题目分析

  • 提示语：“phps源码泄露有时候能帮上忙”
  • 潜台词：出题人告诉你，这道题不需要复杂的漏洞利用（如 SQL 注入、命令注入等），而是需要你通过信息收集，找到一个以 .phps结尾的文件，并从中提取出 Flag。
  • 任务：尝试访问当前页面的 .phps版本，看看是否能直接看到源代码。

3. 具体操作步骤

对当前的 URL 进行修改：

  1. 获取当前 URL： 根据 Host 头，当前的 URL 是： http://8blab47e-e82e-4d75-888c-97621a4db037c.challenge.ctf.show/
  2. 修改文件扩展名： 将 URL 的末尾加上 .phps。

如果当前页面是 index.php： http://8blab47e-e82e-4d75-888c-97621a4db037c.challenge.ctf.show/index.phps

如果 URL 后面已经有文件名（比如 http://.../login.php）：把 .php改成 .phps。

如果 URL 后面没有文件名，通常默认是 index.php： .../index.phps。

  1. 访问并查看： 用浏览器打开这个新 URL。

4. 预期结果

如果运气好，或者出题人真的按套路出牌，：

<?php
// ... 一些 PHP 代码 ...

$flag = "ctfshow{xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx}";

// ... 更多的 PHP 代码 ...
?>

你只需要把 $flag = "...";里面的那串字符复制下来，提交即可。

5. 如果没有找到？

如果访问 .phps返回 404（找不到页面）

  • .bak备份文件： index.php.bak index.bak
  • .swp交换文件（Vim 编辑器生成的）： index.php.swp
  • .git目录泄露： 访问 http://.../.git/，如果能打开，说明可能存在 Git 仓库泄露，可以用工具（如 GitHack）还原整个源码。
  • .zip压缩包： 有时候源码会被打包成 www.zip或 source.zip，访问这些文件可能会下载源码包。

6. 总结

这道题的核心考点是“信息收集”和“对 Web 服务器常见文件结构的了解”。

  • 考的是 .phps源码泄露。

建议：先尝试访问 .../index.phps，如果成功看到源码，直接拿 Flag；如果不行，再考虑其他类型的源码泄露（如 .bak, .swp等）。
